리액트 성능 향상 - 10. 폰트 최적화

위 이미지의 경우, 폰트가 다 다운로드가 되지 않아 폰트가 적용되지 않았습니다.

ttf 폰트가 다운로드가 다 안됐을 때, 폰트 적용이 안됨

폰트 다운로드가 다 완료가 되어야 적용됨

FOUT, FOIT은 브라우저에 따라 커스텀 폰트를 적용하기 전까지의 적용 과정을 말합니다.

어쨌든 두 개 모두 커스텀 폰트가 다운로드가 완전히 되지 않을 경우, 기본 폰트를 보여줍니다.

예를 들면, 처음에는 Noto Sans로 보여주다가, 특정 회사의 폰트가 다운로드 되면 그제서야 그 폰트 스타일을 반영하는 것입니다.

출처: 프론트엔드 성능 최적화 가이드(유동균 지음)

이럴 때 최적화 방법은 다음과 같습니다.

폰트 적용 시점 제어

한마디로 CSS의 font-display 속성으로  FOUT, FOIT 방식 중에 하나를 고르게끔 하는 것입니다.

@font-face {
  font-family: BMYEONSUNG;
  src: url("./assets/fonts/BMYEONSUNG.ttf");

  /* 브라우저 기본 값 */
  font-display: auto;

  /* FOIT(timeout = 3s) */
  font-display: block;

  /* FOUT */
  font-display: swap;

  /*
        FOIT(timeout = 0.1s)
        3초 이후에도 불로오지 못한 경우,
        기본 폰트 유지, 이후 캐시
    */
  font-display: fallback;

  /*
        FOIT(timeout = 0.1s)
        이후 네트워크 상태에 따라
        기본 폰트로 유지할지말지 결정, 이후 캐시
    */
  font-display: optional;
}

폰트 파일 크기 줄이기

transfonter 사이트에서 폰트 확장자를 변경할 수 있습니다.

https://transfonter.org/

Online @font-face generator The @font-face CSS rule allows web developers to specify online fonts to display text on their web pages. By allowing authors to provide their own fonts, @font-face eliminates the need to depend on the limited number of fonts users have installed on their transfonter.org

폰트 파일 크기 비교

다음 순으로 파일 크기가 큽니다.

EOT > TTF/OTF > WOFF > WOFF2

따라서 webp, webm에서 했던대로

woff2, woff 먼저 적용하고, 해당 폰트를 지원하지 않을 경우

ttf를 적용하는 CSS가 필요합니다.

@font-face {
    font-family: BMYEONSUNG;
    src: url('./assets/fonts/BMYEONSUNG.woff2') format('woff2'),
      url('./assets/fonts/BMYEONSUNG.woff') format('woff'),
      url('./assets/fonts/BMYEONSUNG.ttf') format('truetype'),
    font-display: block;
}

폰트 서브셋만 업로드

'KEEP CALM AND RIDE LOGBOARD'에서만 쓰는 폰트를 전체 다운 받을 필요는 없을 것 같습니다.

따라서transfonter 사이트에서 해당 메세지만 담은 서브셋 폰트를 다운받아서 업로드하는 방법도 있습니다.

@font-face {
  font-family: BMYEONSUNG;
  src: url("./assets/fonts/subset-BMYEONSUNG.woff2") format("woff2"), url("./assets/fonts/subset-BMYEONSUNG.woff")
      format("woff");
  font-display: block;
  unicode-range: u+0041, u+0042, u+0043, u+0044, u+0045, u+0047, u+0049, u+004b,
    u+004c, u+004d, u+004e, u+004f, u+0050, u+0052;
}

data-uri 형태의 경우, 파일보다는 확실히 빨리 다운로드 되지만, CSS 파일에 내장되어있기 때문에 큰 파일을 data-uri로 다운해서 import한 경우, CSS 파일의 다운로드가 늦어질 수 있습니다. data-uri를 서브셋으로 만든다면 큰 파일에 비해서는 다운로드가 빨라질 수 있습니다.
